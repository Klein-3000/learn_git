# 配置
位置 `.github/workflows/demo.yml`
```yml
name: my first workflow

# 控制工作流何时运行
on: workflow_dispatch # 允许你从 Actions 标签页手动运行此工作流

# 一次工作流运行由一个或多个作业组成，这些作业可以顺序或并行运行
jobs:
  # 此工作流包含一个名为 "build" 的单一作业
  build:
    # 作业将在其上运行的 Runner 类型
    runs-on: ubuntu-latest

    # 步骤代表作为作业一部分将执行的任务序列
    steps:
      # 使用 Runner 的 shell 运行一组命令
      [-]{一个短杆} name: Run a multi-line script
        run: |
          echo "hello github actions"
          date
 
```
## 并行和串行 与 action
默认是==并行==
```yml
name : learn github action
on : workflows_dispatch
jobs:
	job1:
		runs-on: ubuntu-latest
			steps:
				- name: step1
				  run: |
				  echo jobs-step1
				  date
	job2:
		runs-on: ubuntu-lates
		# =========================
		# 串行的关键
		if: ${{ success() }}
		needs: [job1]
		# ==========================
			steps:
				- name: clone repository
				  # ========================
				  # 克隆并进入本仓库
				  # ========================
				  use: actions/checkout@v3
				  
				- name : run build
				  run :
					  Cargo build --release
				  
							
```
