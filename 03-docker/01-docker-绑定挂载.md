# 绑定挂载

### 1. **权限继承自宿主机**

- 容器内挂载点的 **用户/组 ID、读写执行权限（rwx）** 与宿主机目录**完全一致**。
- Docker **不会修改、转换或覆盖**这些权限。
- 容器内的进程以某个 UID/GID 运行（如 `root` = UID 0），能否访问挂载目录，取决于：
    
    > **该 UID/GID 在宿主机上对该目录是否有权限**
    
### 2. **SELinux（Fedora/RHEL/CentOS）会额外拦截**

- 即使文件权限允许，**SELinux 默认阻止容器访问普通宿主机目录**。
- 表现为：`Permission denied`，即使 `root` 也无法访问。
```shell
# :Z — 私有卷（仅当前容器使用）
docker run -v /host/path:/container/path:Z ...

# :z — 共享卷（多个容器共享）
docker run -v /host/path:/container/path:z ...
```
✅ 这会自动为目录打上 `container_file_t` 标签，放行访问。
### 3. **挂载空目录 ≠ 初始化内容**

- 如果宿主机目录为空（或新建），**容器内原路径的内容会被隐藏**，不会复制过来。
- 容器看到的就是一个**空目录**，且权限由宿主机决定（通常是创建者权限）。

> ⚠️ 常见陷阱：用空宿主机目录挂载到数据库数据目录（如 `/var/lib/mysql`），导致数据库启动失败（因为“数据丢失”）。
