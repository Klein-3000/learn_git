# 资源限制
Docker 通过 **`docker run`** 或 **`docker create`** 的参数来设置容器的资源限制，底层依赖 Linux 的 **cgroups（Control Groups）** 机制。主要分为以下几类：
## 1. CPU
| 参数                            | 说明                                | 示例                               |
| ----------------------------- | --------------------------------- | -------------------------------- |
| `--cpus=<value>`              | 限制容器最多使用多少个 CPU 核心（Docker 1.13+）  | `--cpus=1.5`（最多用 1.5 个核）         |
| `--cpu-period=<microseconds>` | 设置 CPU 调度周期（通常与 `--cpu-quota` 配合） | `--cpu-period=100000`            |
| `--cpu-quota=<microseconds>`  | 在一个周期内允许使用的 CPU 时间（微秒）            | `--cpu-quota=50000` → 相当于 0.5 核  |
| `--cpuset-cpus=<cpu-list>`    | 绑定容器到特定 CPU 核心                    | `--cpuset-cpus=0,2`（只用 0 和 2 号核） |
| `--cpu-shares=<weight>`       | 设置 CPU 权重（相对值，默认 1024）            | `--cpu-shares=512`（优先级减半）        |
## 2. 内存
| 参数                             | 说明                               | 示例                               |
| ------------------------------ | -------------------------------- | -------------------------------- |
| `-m` 或 `--memory=<bytes>`      | 限制容器最大可用内存                       | `--memory=512m`                  |
| `--memory-swap=<bytes>`        | 限制 内存 + swap 总量（必须 ≥ `--memory`） | `--memory=512m --memory-swap=1g` |
| `--memory-reservation=<bytes>` | 软限制（当系统内存紧张时才生效）                 | `--memory-reservation=256m`      |
| `--oom-kill-disable`           | 禁用 OOM Killer（不推荐）               | `--oom-kill-disable`             |
| `--oom-score-adj`              | 调整 OOM 优先级（-1000 ~ 1000）         | `--oom-score-adj=500`            |
⚠️ 注意：如果只设 `--memory`，`--memory-swap` 默认等于 `--memory`（即 **禁用 swap**）。
## 3. Block(磁盘I/O)
|参数|说明|示例|
|---|---|---|
|`--blkio-weight`|设置 I/O 权重（10–1000，默认 500）|`--blkio-weight=300`|
|`--blkio-weight-device`|对特定设备设置权重|`--blkio-weight-device=/dev/sda:200`|
|`--device-read-bps`|限制设备读取速率（字节/秒）|`--device-read-bps=/dev/sda:1mb`|
|`--device-write-bps`|限制设备写入速率|`--device-write-bps=/dev/sda:1mb`|
|`--device-read-iops` / `--device-write-iops`|限制 IOPS|`--device-write-iops=/dev/sda:100`|
## 总结

| 问题     | 答案                                                                   |
| ------ | -------------------------------------------------------------------- |
| 资源限制命令 | `--cpus`, `--memory`, `--blkio-weight`, `--pids-limit`, `--ulimit` 等 |
| 默认是否限制 | 否！容器默认可耗尽宿主机所有资源                                                     |
| 最佳实践   | 生产环境必须显式设置 CPU 和内存限制，避免“坏邻居”问题                                       |
# 案例
```bash
# 内存限制
docker  run [--memory 500M]{最多 500m 内存} [--memory-reservation 200M]{**内存不足时**,才有效果,确保一定有 200M 内存可用} alpine
```
